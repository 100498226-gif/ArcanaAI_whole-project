### Phase 7.5 — Cursor UI (PR #7, merged to main)

**`cursor/src/api/arcanaClient.ts`** — 5 new API functions + 4 typed interfaces:
- `checkPendingReview()` — `GET /admin/updater/review/pending`; returns null silently on 403/error (non-admin users)
- `fetchWeeklyReview()` — `GET /admin/updater/review?week=&year=`
- `fetchUpdateHistory()` — `GET /admin/updater/history?week=&year=&page_size=100`
- `revertUpdate()` — `POST /admin/updater/revert/{id}`; error detail handles both string and list (Pydantic 422) formats
- `acknowledgeReview()` — `POST /admin/updater/review/{id}/acknowledge`; checks `response.ok`

**`cursor/src/sidebar/SidebarProvider.ts`**:
- Review badge added to HTML header with `aria-label`
- Health ping now also calls `checkPendingReview` every 60s; uses `_reviewIsPending` flag to only send `reviewClear` on transition (not every 60s)
- `_handleFetchReview` — fetches review + history in parallel; guarded against undefined week/year
- `_handleRevertUpdate` — calls backend, sends result back to webview
- `_handleAcknowledgeReview` — clears all pending state on success; non-critical failure self-corrects on next ping

**`cursor/webview/main.js`**:
- `showReviewBadge()` — resets `disabled` + title (badge can never get stuck)
- `hideReviewBadge()` — adds `hidden` class
- `renderReviewCard()` — renders narrative (marked.js), patterns, per-change rows; duplicate card prevention (scrolls to existing); empty-updates empty state
- `renderChangeItem()` — significant/high-risk colour borders, recommended-review flags, status badge, Revert button
- `openRevertForm()` — inline textarea with visible 20-char validation div (replaces unreliable `setCustomValidity`); form stays open on error for retry
- `handleRevertResult()` — success: removes form, adds badge, shows note; error: re-enables buttons, shows error inside form, clears previous error on retry

**`cursor/webview/styles.css`** — new styles: `.review-badge`, `.review-card`, `.change-list` (max-height 420px), `.change-item` (significant/high-risk borders), `.revert-form`, `.revert-validation-msg`, `.change-list-empty`, `.system-message`

**`cursor/src/test/suite/components.test.ts`** — 24 new tests (52 total, up from 28):
- `buildHighRiskSet`, `buildRecommendedMap`, `changeTypeIcon`, `isReverted`
- `applyReviewPending` / `applyReviewClear` state transitions
- `isCorrectionValid` (all edge cases: empty, whitespace, too-short, exactly-20, long)
- `shouldShowBadge` (pending=true, pending=false, null, review_id=null)
